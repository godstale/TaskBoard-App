## Project Overview

TaskBoard App is an **Electron 33 + React 18 desktop app** for visualizing [TaskOps](https://github.com/godstale/TaskOps) projects.
It reads `taskops.db` (SQLite) generated by TaskOps in read-only mode and renders the Epic/Task hierarchy, operation history, and resources.

## Architecture Summary

### Data Flow

```
taskops.db (SQLite, read-only)
  └── src/core (better-sqlite3)
        └── src/main (Electron main process)
              ├── IPC channels
              └── src/renderer (React 18)
                    └── window.taskboard (context-bridge)
```

### Key Source Files

| File | Purpose |
|------|---------|
| `src/core/models.ts` | TypeScript interfaces for all domain types |
| `src/core/db.ts` | `openDb` / `closeDb` (better-sqlite3) |
| `src/core/queries.ts` | All read-only query functions |
| `src/core/watcher.ts` | chokidar watcher + polling fallback |
| `src/main/index.ts` | Electron main process |
| `src/main/ipc.ts` | IPC channel name constants |
| `src/main/preload.ts` | Context-bridge (`window.taskboard`) |
| `src/renderer/App.tsx` | React root + screen routing |
| `src/renderer/useTaskBoard.ts` | Renderer state hook (IPC) |

### Domain Types

```typescript
type TaskStatus    = 'todo' | 'in_progress' | 'interrupted' | 'done' | 'cancelled'
type TaskType      = 'project' | 'epic' | 'task' | 'objective'
type OperationType = 'start' | 'progress' | 'complete' | 'error' | 'interrupt'
type ResourceType  = 'input' | 'output' | 'reference' | 'intermediate'
```

### Query Functions (`src/core/queries.ts`)

| Function | Returns |
|----------|---------|
| `getProject(db)` | Root project row |
| `getEpicsWithTasks(db)` | Nested Epic → Task → SubTask tree |
| `getWorkflowOrder(db)` | Tasks sorted by `seq_order` |
| `getOperations(db, taskId?)` | All or per-task operations |
| `getResources(db, taskId?)` | All or per-task resources |
| `getSettings(db)` | All settings rows |
| `getProjectList(root)` | Subdirs of `root` containing `taskops.db` |
| `getProjectSummary(db)` | Aggregated task counts by status |

### Electron IPC Channels (`src/main/ipc.ts`)

| Channel | Direction | Purpose |
|---------|-----------|---------|
| `get-project-list` | renderer → main | Scan root for projects |
| `get-all-data` | renderer → main | Load DB snapshot + start watcher |
| `select-folder` | renderer → main | Open OS folder picker |
| `db-changed` | main → renderer | Notify renderer to reload |

## Coding Rules

1. **Read-only**: Never write to `taskops.db`. All queries are SELECT only.
2. **IPC boundary**: Renderer never touches the filesystem or SQLite directly — everything goes through `window.taskboard`.
3. **TypeScript strict**: Avoid `any`. Strict TypeScript 5.4.
4. **Testing**: Add/update Vitest unit tests for core/renderer changes; Playwright for UI flows.
